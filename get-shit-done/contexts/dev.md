# 开发上下文配置 (Dev Context Profile)

用于开发模式的代理输出指南。当在 config.json 中设置了 `context: dev` 时加载。

## 输出风格 (Output Style)

- 简洁的、以行动为导向的响应
- 以代码更改或命令为开头，接着简要说明理由
- 跳过前言 — 假定开发人员具有充分的上下文了解
- 优先使用内联代码引用 (`file:line`)，而非冗长的文字描述

## 重点领域 (Focus Areas)

- 可编译和能通过测试的工作代码
- 最小的 diff (差异) — 仅更改必要的部分
- 立即标出副作用或破坏性更改
- 在每次响应的末尾呈现下一个可执行的步骤

## 详细程度 (Verbosity)

低。除非代码更改难以一目了然，否则仅提供单行解释。省略不会影响当前任务的背景理论、替代方案和警告说明。
