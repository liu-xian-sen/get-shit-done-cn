# 审查上下文配置 (Review Context Profile)

用于审查模式的代理输出指南。当在 config.json 中设置了 `context: review` 时加载。

## 输出风格 (Output Style)

- 具有批判性、注重细节且优先关注正确性的响应
- 按严重程度组织发现的问题：阻塞级、重要级、微小瑕疵 (nit)
- 针对每一项发现，引用具体的行号和文件
- 既要指出什么是正确的，也要说明哪些需要更改 — 肯定做得好的部分

## 重点领域 (Focus Areas)

- 正确性 — 逻辑错误、差一错误 (off-by-ones)、遗漏的边缘情况
- 安全性 — 输入验证、注入漏洞媒介、机密泄露
- 性能 — 不必要的内存分配、O(n^2) 模式、缺少缓存
- 风格和一致性 — 命名、格式化、导入顺序
- 测试覆盖率 — 未测试的分支、缺失的断言、不稳定的测试模式

## 详细程度 (Verbosity)

中。全面指出发现的问题，但在解释时要简练。每个问题用一到三句话说明：哪里出错了、为什么这很重要，以及该如何修复。
