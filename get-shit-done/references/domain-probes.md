# 领域感知探测模式

供 `/gsd-begin`、`/gsd-discuss-phase` 和领域探索工作流共享的参考文档。

当用户提到某个技术领域时，使用这些探针提出有见解的追问。不要将其作为清单逐一过一遍——根据上下文选择 2-3 个最相关的问题。目标是揭示用户可能尚未考虑的隐藏假设和权衡。

---

## 身份验证

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "登录"或"认证" | OAuth（哪个提供商？）、JWT 还是基于会话？需要社交登录还是只需邮箱/密码？ |
| "用户"或"账户" | 需要 MFA 吗？密码重置流程？邮箱验证？ |
| "会话" | 会话时长和刷新策略？服务端会话还是无状态 token？ |
| "角色"或"权限" | RBAC、ABAC 还是简单角色检查？有多少种不同角色？ |
| "API 密钥" | 密钥轮换策略？每个密钥的范围权限？每个密钥的速率限制？ |

---

## 实时更新

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "实时"或"实时更新" | WebSockets、SSE 还是轮询？哪些内容需要真正实时，哪些可以最终一致？ |
| "通知" | 推送通知（浏览器/移动端）、纯应用内，还是两者？持久化和已读回执？ |
| "协作"或"多人" | 冲突解决策略？操作变换还是 CRDT？预期并发用户数？ |
| "聊天"或"消息" | 消息历史和搜索？输入指示器？已读回执？ |
| "流式传输" | 重连策略？连接断开时——队列还是丢弃？ |

---

## 仪表盘

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "仪表盘" | 数据来源是什么？有多少种不同视图？ |
| "图表"或"图形" | 交互式还是静态？支持下钻？导出为 CSV/PDF？ |
| "指标"或"KPI" | 刷新策略——实时、周期性轮询还是按需？可接受的数据延迟？ |
| "管理面板" | 基于角色的可见性？除查看外还有哪些操作（编辑、删除、审批）？ |
| "移动"或"响应式" | 简化的移动视图还是完全一致？图表的触控交互？ |

---

## API 设计

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "API" | REST、GraphQL 还是 RPC 风格？仅内部使用还是公开？ |
| "端点"或"路由" | 版本策略（URL 路径、请求头、查询参数）？破坏性变更政策？ |
| "分页" | 基于游标还是偏移量？预期结果集大小？稳定排序保证？ |
| "速率限制" | 按用户、按 IP 还是按 API 密钥？突发允许量？如何向客户端传达限制？ |
| "错误" | 结构化错误格式？错误码 vs 错误消息？生产中错误详情的详细程度？ |

---

## 数据库

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "数据库"或"存储" | SQL 还是 NoSQL？选择的驱动因素——关系完整性、灵活性还是规模？ |
| "ORM"或"查询" | ORM（哪个？）还是原生查询？查询构建器作为中间方案？ |
| "迁移" | 迁移工具？回滚策略？如何处理数据迁移 vs schema 迁移？ |
| "种子数据"或"测试数据" | 开发用种子数据？真实感伪数据还是最小固定数据？ |
| "规模"或"性能" | 读写比率？读副本？连接池策略？ |

---

## 搜索

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "搜索" | 全文还是精确匹配？专用搜索引擎（Elasticsearch、Meilisearch）还是数据库级别？ |
| "过滤"或"分面" | 分面过滤？过滤维度数量？组合过滤器（AND/OR）？ |
| "自动完成"或"预输入" | 防抖策略？最小字符阈值？结果排名？ |
| "索引" | 索引大小和更新频率？实时索引还是批量？可接受的索引延迟？ |
| "模糊"或"拼写容错" | 模糊匹配？同义词支持？语言特定的词干分析？ |

---

## 文件上传/存储

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "上传"或"文件上传" | 本地文件系统还是云端（S3、GCS、Azure Blob）？直接上传还是通过服务器？ |
| "图片"或"媒体" | 处理管道——调整大小、压缩、缩略图生成？格式转换？ |
| "大小限制" | 最大文件大小？每用户最大总存储量？超出限制时如何处理？ |
| "CDN" | CDN 分发？更新文件的缓存失效？访问控制的签名 URL？ |
| "文档"或"附件" | 病毒扫描？预览生成？上传文件的版本控制？ |

---

## 缓存

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "缓存"或"性能" | 在哪里缓存——浏览器、CDN、应用层、数据库查询缓存？ |
| "失效" | 失效策略——TTL、事件驱动还是手动？旁路缓存 vs 写穿缓存？ |
| "过时数据" | 可接受的数据新鲜度窗口？过时重验证模式？ |
| "Redis"或"Memcached" | 缓存拓扑——单节点还是集群？需要持久化还是纯缓存？ |
| "CDN"或"边缘" | 静态资源的边缘缓存？边缘动态内容？缓存键策略？ |

---

## 测试

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "测试" | 单元测试、集成测试和 E2E 的比例？在哪里投入最多测试精力？ |
| "mock"或"存根" | Mock 外部服务还是使用测试容器？数据库 mock 策略？ |
| "CI"或"流水线" | 测试在 CI 中运行？并行测试执行？PR 时测试还是推送时测试？ |
| "覆盖率" | 覆盖率目标？覆盖率作为门禁还是建议？哪些指标（行、分支、函数）？ |
| "E2E"或"浏览器测试" | Playwright、Cypress 还是其他？有头还是无头？视觉回归测试？ |

---

## 部署

| 用户提到 | Agent 用领域知识探测 |
|---|---|
| "部署"或"托管" | 容器、无服务器还是传统 VM/VPS？托管平台（Vercel、Railway）还是自托管？ |
| "CI/CD"或"流水线" | GitHub Actions、GitLab CI 还是其他？合并到主分支时部署还是手动触发？ |
| "环境" | 有多少个环境（dev、staging、prod）？环境一致性策略？ |
| "回滚" | 回滚策略？蓝绿、金丝雀还是即时回滚？数据库回滚考虑？ |
| "密钥"或"配置" | 密钥管理——环境变量、vault 还是平台原生？每环境的配置策略？ |
