# 常见 Bug 模式

在形成假设之前需要扫描的高频 bug 模式清单。按频率排序。**优先检查这些**——它们覆盖了所有技术栈约 80% 的 bug。

<patterns>

## 空值 / 未定义访问

- **空属性访问** —— 在 `null` 或 `undefined` 上访问属性，缺少空值检查或可选链
- **缺少返回值** —— 函数返回 `undefined` 而非预期值，缺少 `return` 语句或走错了分支
- **解构空值** —— 对 `null`/`undefined` 进行数组/对象解构，API 返回的是错误形状而非数据
- **未设置默认的可选参数** —— 可选参数未设默认值就使用，调用方省略了参数

## 差一错误 / 边界

- **错误的循环边界** —— 循环从 1 开始而不是 0，或在 `length` 处结束而不是 `length - 1`
- **栅栏柱错误** —— "N 个项目需要 N-1 个分隔符"计数错误
- **包含 vs 排除** —— 范围边界 `<` 与 `<=`，slice/substring 的结束索引
- **空集合** —— `.length === 0` 落入假设存在元素的逻辑

## 异步 / 时序

- **缺少 await** —— 调用异步函数时没有 `await`，得到 Promise 对象而非解析值
- **竞态条件** —— 两个异步操作在没有协调的情况下读写同一状态
- **过时闭包** —— 回调捕获了旧的变量值，而非当前值
- **初始化顺序** —— 事件处理程序在设置完成前触发
- **泄漏计时器** —— 超时/间隔未清理，在组件/上下文销毁后触发

## 状态管理

- **共享变更** —— 就地修改的对象/数组影响其他消费者
- **过时渲染** —— 状态已更新但 UI 未重新渲染，缺少响应式触发器或引用错误
- **处理程序中的过时状态** —— 闭包在绑定时捕获状态，而非当前值
- **双重事实来源** —— 同一数据存储在两处，其中一处不同步
- **无效转换** —— 状态机允许缺少守卫条件的转换

## 导入 / 模块

- **循环依赖** —— 模块 A 导入 B，B 导入 A，其中一个得到 `undefined`
- **导出不匹配** —— 默认导出与命名导出，`import X` 与 `import { X }`
- **扩展名错误** —— `.js` vs `.cjs` vs `.mjs`，`.ts` vs `.tsx`
- **路径大小写敏感** —— 在 Windows/macOS 上工作，在 Linux 上失败
- **缺少扩展名** —— ESM 要求在导入中明确指定文件扩展名

## 类型 / 强制转换

- **字符串 vs 数字比较** —— `"5" > "10"` 为 `true`（字典序），`5 > 10` 为 `false`
- **隐式强制转换** —— 使用 `==` 而不是 `===`，truthy/falsy 意外（`0`、`""`、`[]`）
- **数值精度** —— `0.1 + 0.2 !== 0.3`，大整数精度丢失
- **假值的合法值** —— 值为 `0` 或 `""`，合法但为假值

## 环境 / 配置

- **缺少环境变量** —— 环境变量缺失或在 dev/prod/CI 中值不同
- **硬编码路径** —— 在一台机器上工作，在另一台上失败
- **端口冲突** —— 端口已被占用，上一个进程仍在运行
- **权限拒绝** —— 部署中使用了不同的用户/组
- **缺少依赖** —— 未在 package.json 中或未安装

## 数据形状 / API 契约

- **响应形状变更** —— 后端已更新，前端期望旧格式
- **容器类型错误** —— 期望对象但得到数组或反之，`data` vs `data.results` vs `data[0]`
- **缺少必填字段** —— 载荷中省略了必填字段，后端返回验证错误
- **日期格式不匹配** —— ISO 字符串 vs 时间戳 vs 本地化字符串
- **编码不匹配** —— UTF-8 vs Latin-1，URL 编码，HTML 实体

## 正则 / 字符串

- **粘性 lastIndex** —— 带 `g` 标志的正则用 `.test()` 后再 `.exec()`，调用之间未重置 `lastIndex`
- **缺少转义** —— `.` 匹配任意字符，`$` 是特殊字符，反斜杠需要双写
- **贪婪过匹配** —— `.*` 吃掉分隔符，需要用 `.*?`
- **引号类型错误** —— 字符串插值需要反引号用于模板字面量

## 错误处理

- **吞噬错误** —— 空的 `catch {}` 或只记录日志不重新抛出/处理
- **错误类型错误** —— 捕获基类 `Error` 而非需要的特定类型
- **处理程序中的错误** —— 清理代码抛出异常，掩盖了原始错误
- **未处理的 rejection** —— 缺少 `.catch()` 或 `await` 周围没有 try/catch

## 作用域 / 闭包

- **变量遮蔽** —— 内层作用域声明了同名变量，隐藏外层变量
- **循环变量捕获** —— 所有闭包共享同一个 `var i`，使用 `let` 或绑定
- **丢失 this 绑定** —— 回调丢失上下文，需要 `.bind()` 或箭头函数
- **作用域混淆** —— `var` 提升到函数级，`let`/`const` 是块级作用域

</patterns>

## 如何使用此清单

1. **在形成任何假设之前**，根据症状扫描相关类别
2. **将症状与模式匹配** —— 如果 bug 涉及"undefined is not an object"，先检查空值/未定义
3. **每个已检查的模式都是假设候选** —— 用证据验证或排除
4. **如果没有模式匹配**，进行开放式调查

### 症状到类别快速映射

| 症状 | 优先检查 |
|---------|------------|
| "Cannot read property of undefined/null" | 空值/未定义访问 |
| "X is not a function" | 导入/模块、类型/强制转换 |
| 有时正常，有时失败 | 异步/时序、状态管理 |
| 本地正常，CI/生产失败 | 环境/配置 |
| 显示错误数据 | 数据形状、状态管理 |
| 差一项 / 缺少最后一项 | 差一错误/边界 |
| "Unexpected token" / 解析错误 | 数据形状、类型/强制转换 |
| 内存泄漏 / 资源增长 | 异步/时序（清理）、作用域/闭包 |
| 无限循环 / 最大调用栈 | 状态管理、异步/时序 |

</usage>
